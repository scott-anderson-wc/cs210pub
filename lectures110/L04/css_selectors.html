CSS Selectors

When we talked about CSS rules, we described the generic form of a rule as in Figure 1. At the head of the rule is the selector, followed by a series of property and values pairs enclosed in curly braces.

Today's focus is on the selector. The selector indicates which element(s) the rule will apply to. So far, we have seen a few kinds of selectors and these notes will add some more to this list.

  • element , e.g., p, em, li, etc. Such a selector will apply to all elements of this kind in the document.
  • element.classname, e.g, span.booktitle. Such a selector will apply to the indicated elements with the attribute class=classname.
  • .classname, e.g., .tiltright, .tiltleft. Such a selector will apply to different kinds of elements that share the attribute class=classname.
  • #idName, e.g., #main, etc. Such a selector applies to a single element that has the attribute id=idName.
  • element, element, ..., e.g., h1, h2, h3, etc. Such a selector is used in those cases we want elements of different kinds to share a set of rules. This way, we avoid retyping the same rules. It is also allowed to add individual rules for such elements separately as well.

One could certainly stop here and don't worry about more kinds of selectors, because by adding class or id attributes to our elements, we could always use one of the above described selectors. The problem with the approach of using styling hooks is that it makes the code difficult to read and maintain, since we will be adding lots of class and id attributes only for the purpose of styling (hence the name: styling hooks).

To illustrate, here is an example (from the "Book of CSS3"):

<ul>
    <li class="first odd"><span>F</span>irst list item</li> 
    <li>Second item</li> 
    <li class="odd">Third item</li> 
    <li class="last">Last item</li> 
</ul>

Notice that it's possible to have multiple class names in a single class attribute (first and odd). However, imagine a situation in which you add more items to this list, you'll have to reorder the class attributes, because the sequence of odd and even items might change, etc.

There are two ways to deal with this situation, which we describe in the following sections: descendant selectors and pseudo-selectors.

Descendant Selectors

As we will keep repeating throughout this semester, an HTML document can be modeled as a tree. At the root is the <html> element, with all other elements as descendants, grouped in different families. Chrome's Inspect Element gives a very good depiction of such a tree (we will call this depiction a DOM (Document Object Model), and will study it in more detail in the future).

Here is a graphic of the DOM tree from the example you worked in Lab.

Since DOM is a tree, we will use the same analogy with the family tree that we used when talking about folders and files in creating relative URLs. Nodes (that is elements), can be ancestors or descendants, parents or children, as well as siblings.

As you already know, if you apply a rule to an element (for example, removing the underline for <a> element, every link in the document will be styled without an underline. But this is not what you may intend. You might want that only links in the navbar don't have an underline, while all the others should.

The solution in this case is a descendant selector. Such a selector specifies the path inside the DOM tree for the element who want the rule to apply exclusively.

Figure 3 shows the path of elements in which our a element is nested, as displayed by Chrome's Inspect Element window. This is an absolute path, starting from the root, html. When we specify the selector, we can use only parts of such a path, such as nav ul li a, or simply nav a.

Keep in mind that in order to use the descendant selectors, you need to use more structure in your HTML document. If all your links are directly in the body or unamed divs, there is no unique path to different elements, since they are all siblings.

In addition to the example we saw, there are other selectors that use the DOM relationships. For example, span>a, which selects all a elements that have as a parent a span, or div+h3 will select h3 elements that immediately follow a div (an adjacent sibling), and many more.

To better understand the different kinds of CSS selectors, refer to the code examples in W3Schools CSS Selectors' page.

Pseudo-selectors: pseudo-class and pseudo-element

The selectors we have seen so far can be grouped under the label of DOM selectors, because they operate with information that is explicit in the DOM tree (element names or their attributes). However, in many cases, we would like to style aspects of the elements that cannot be found directly in the tree, for example, how a link should look like when we hover over it. To do this, we will use pseudo-selectors. Such selectors act on elements or information that sits outside of the document tree (such as the state of a link, the first letter of a paragraph or the last child of a parent element).

Pseudo-selectors come in two flavors: pseudo-classes and pseudo-elements.

Examples of using states of an element are: information about link states: :hover, :visited, :active, about form inputs states: :checked, :disabled. Examples of using the type of an element are structural pseudo-classes such as :first-child [see example], :nth-child(odd) [see example], :nth-of-type(n) [see example]. Finally, in CSS3, there is a new pseudo-class known as :not, which negates a rule for the selected element (that is, the rule will apply to all other elements that are not selected). See example.
  • A pseudo-element provides access to an elementâ€™s subpart, which includes those pseudo-elements that select portions of text nodes, for instance, :first-line [see example] or :first-letter [see example].

In CSS3, to differentitate between pseudo-classes and pseudo-elements, the syntax to refer to them has changed. An element is referred with a double colon. You will see both examples on the Web, because they are still supported by browsers.
