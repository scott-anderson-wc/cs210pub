  2. I'm somewhat confused about what the "all and" part of the @media line stand for.

    You can customize the kind of device (e.g. print versus screen). That goes where the all is. See MDN media queries

  3. In a media query, how do you specify mobile layouts depending on horizontal/vertical orientation?

    Use (orientation: landscape) or (orientation: portrait). Again, See MDN media queries

  4. How should you know what pixel size to specify in a media query?

    There are standard sizes/breakpoints for different common devices. Chrome developer tools can help here. Your book lists some.

  5. What is the difference between layout viewport and visual viewport other than size?

    The former is used, as you might guess, for layout. The latter is a window on the former, so you might need to scroll, magnify, etc.

  6. Why is the visual viewport narrower than the layout viewport?

    Because we either want to see everything or part of everything, but we never want to see more than everything.

  7. what is a breakpoint?

    A device size that triggers a change in the CSS rules (and, often, a change in layout).

  8. I noticed that webpages use media queries for mobile devices, but some webpages use a separate page for mobile devices. How can the webpage detect that the user is viewing the page from mobile device? Also, why do some webpages use a separate mobile page instead of using media queries?

    The browser sends an identifying string saying "I'm Chrome" or "I'm Firefox" and your browser can detect that. Best practice is not to do that, though, because (1) it's better to think of capabilities, (2) you might not catch every case, and (3) browsers can lie.

    Separate pages can allow the CSS to be streamlined, omitting anything that is irrelevant, instead of overriding it, but in many cases the separate page is a legacy from the days before media queries.

  9. What counts and does not count as "responsive"? When we use ems and set widths as percentages, so that they adjust as the window expands, is that also responsive, or is there a different term?

    Ems and percentages is part of being responsive. Being responsive doesn't have a mathematical definition, but is the goal of adjusting (responding) to different size devices.

  10. Difference between media query css and normal css when it comes to rendering hierarchy. Is it guaranteed that putting a certain size limit on the alternate css will make it disable at the right size? Also, which version of the css gets priority, the regular version or the one that specifies a certain window size?

    There's no difference. Later rules override earlier rules, so standard practice is to specify the default first, and then override for larger devices. Thus, media queries don't complicate the rules for how CSS cascades. As for the guarantee, a browser could be broken, non-compliant or just old, but there's nothing we can do about that.
